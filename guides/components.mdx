APM Enterprise provides a small set of reusable components that handle layout, data display, forms, and feedback. All components use Tailwind CSS for styling and the `cn` utility for class composition.

<Tip>
The `cn` utility at `src/utils/cn.js` combines `clsx` and `tailwind-merge`. Use it to merge conditional and overridden Tailwind classes without class conflicts:

```js
import { clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs) {
    return twMerge(clsx(inputs));
}
```

Pass any mix of strings, arrays, and objects. `twMerge` resolves conflicts between Tailwind utility classes so the last-specified value wins.
</Tip>

<Note>
The `money` formatter at `src/utils/formatters.js` formats a number as Peruvian sol currency using the browser's `Intl.NumberFormat` API:

```js
export function money(n) {
    try {
        return new Intl.NumberFormat("es-PE", { style: "currency", currency: "PEN" }).format(n);
    } catch {
        return "S/" + n;
    }
}
```

`ProjectList` uses `money(p.budget)` to display each project's budget as a formatted currency string (e.g. `S/ 6,000.00`).
</Note>

## Button

A styled `<button>` element with variant and size props. Accepts an optional `icon` node that renders before the label.

**Props**

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | string | `"primary"` | `primary`, `secondary`, `outline`, `ghost`, `danger` |
| `size` | string | `"md"` | `sm`, `md`, `lg` |
| `icon` | ReactNode | — | Icon element rendered before children |
| `className` | string | — | Additional Tailwind classes |
| `...props` | — | — | All native `<button>` attributes |

<AccordionGroup>
  <Accordion title="Button usage examples">

  </Accordion>
  <Accordion title="Button source code">

```jsx
import React from "react";
import { cn } from "../utils/cn";

export function Button({
    children,
    variant = "primary",
    size = "md",
    className,
    icon,
    ...props
}) {
    const variants = {
        primary: "bg-primary text-white shadow-lg shadow-primary/20 hover:bg-primary/90",
        secondary: "bg-white text-obsidian border border-muted/30 hover:bg-muted/10",
        outline: "bg-transparent border-2 border-primary text-primary hover:bg-primary/5",
        ghost: "bg-transparent text-muted-foreground hover:bg-muted/30 hover:text-obsidian",
        danger: "bg-red-500 text-white shadow-lg shadow-red-500/20 hover:bg-red-600",
    };

    const sizes = {
        sm: "px-3 py-1.5 text-xs rounded-xl",
        md: "px-5 py-2.5 text-sm rounded-2xl",
        lg: "px-7 py-3.5 text-base rounded-3xl",
    };

    return (
        <button
            className={cn(
                "inline-flex items-center justify-center gap-2 font-bold transition-all active:scale-95 disabled:opacity-50 disabled:pointer-events-none",
                variants[variant],
                sizes[size],
                className
            )}
            {...props}
        >
            {icon && <span className="flex-shrink-0">{icon}</span>}
            {children}
        </button>
    );
}
```

## TextField

A labeled text input with inline error display and focus ring styling.

**Props**

| Prop | Type | Default | Description |
|---|---|---|---|
| `label` | string | — | Label text rendered above the input |
| `value` | string | — | Controlled input value |
| `onChange` | `(value: string) => void` | — | Called with the raw string value (not the event) |
| `placeholder` | string | — | Input placeholder text |
| `type` | string | `"text"` | HTML input type (`text`, `password`, `email`, etc.) |
| `error` | string | — | Validation error message shown below the input |
| `className` | string | — | Additional Tailwind classes on the wrapper |

<Tip>
`onChange` receives the string value directly — not the `React.ChangeEvent`. You do not need to call `e.target.value` in your handler.
</Tip>

```jsx
import React from "react";
import { cn } from "../utils/cn";

export function TextField({ label, value, onChange, placeholder, error, className, type = "text" }) {
    const id = `field-${label?.toLowerCase().replace(/\s+/g, '-')}`;

    return (
        <div className={cn("flex flex-col gap-2", className)}>
            {label && (
                <label htmlFor={id} className="text-sm font-medium text-muted-foreground px-1">
                    {label}
                </label>
            )}
            <input
                id={id}
                type={type}
                value={value}
                onChange={(e) => onChange(e.target.value)}
                placeholder={placeholder}
                className={cn(
                    "w-full rounded-2xl bg-white px-5 py-3 text-sm shadow-sm border-2 border-transparent transition-all outline-none",
                    "placeholder:text-muted-foreground/60",
                    "focus:border-primary/20 focus:ring-4 focus:ring-primary/10",
                    error ? "border-red-500/20 bg-red-50/50 focus:ring-red-500/10" : "hover:bg-muted/10",
                    className
                )}
            />
            {error && (
                <p className="px-2 text-xs font-bold text-red-500 animate-in fade-in slide-in-from-top-1">
                    {error}
                </p>
            )}
        </div>
    );
}
```

## Select

A custom dropdown component with a click-outside handler. Renders a styled button trigger and an animated dropdown list.

**Props**

| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | string | — | Currently selected option value |
| `onChange` | `(value: string) => void` | — | Called with the selected option's value |
| `options` | `{ value: string, label: string }[]` | — | Array of selectable options |
| `label` | string | — | Label text rendered above the trigger |
| `className` | string | — | Additional Tailwind classes on the wrapper |

<AccordionGroup>
  <Accordion title="Select usage examples">

```jsx

        { value: "paused", label: "Pausado" },
    ]}

// Project status field in the create form

    label="Estado del Proyecto"
    value={status}
    onChange={setStatus}
```

  </Accordion>
  <Accordion title="Select source code">

```jsx
import React, { useState, useRef, useEffect } from "react";
import { ChevronDown } from "lucide-react";
import { cn } from "../utils/cn";

export function Select({ value, onChange, options, label, className }) {
    const [isOpen, setIsOpen] = useState(false);
    const containerRef = useRef(null);

    const selectedOption = options.find((opt) => opt.value === value) || options[0];

    useEffect(() => {
        function handleClickOutside(event) {
            if (containerRef.current && !containerRef.current.contains(event.target)) {
                setIsOpen(false);
            }
        }
        document.addEventListener("mousedown", handleClickOutside);
        return () => document.removeEventListener("mousedown", handleClickOutside);
    }, []);

    return (
        <div className={cn("flex flex-col gap-2.5", className)} ref={containerRef}>
            {label && (
                <span className="text-[10px] font-medium uppercase tracking-widest text-muted-foreground px-1">
                    {label}
                </span>
            )}
            <div className="relative">
                <button
                    type="button"
                    onClick={() => setIsOpen(!isOpen)}
                    className={cn(
                        "flex w-full items-center justify-between rounded-2xl bg-white px-6 py-4 text-sm font-bold shadow-sm ring-1 ring-muted/20 transition-all focus:ring-4 focus:ring-primary/10",
                        isOpen && "ring-primary/30 shadow-md"
                    )}
                >
                    <span className="text-obsidian tracking-wide">
                        {selectedOption.label}
                    </span>
                    <ChevronDown
                        size={16}
                        className={cn("text-primary transition-transform duration-300", isOpen && "rotate-180")}
                    />
                </button>

                {isOpen && (
                    <div className="absolute top-full right-0 z-50 mt-2 min-w-full overflow-hidden rounded-2xl border border-muted/10 bg-white shadow-2xl animate-in fade-in zoom-in-95 duration-200">
                        {options.map((option) => (
                            <button
                                key={option.value}
                                type="button"
                                onClick={() => {
                                    onChange(option.value);
                                    setIsOpen(false);
                                }}
                                className={cn(
                                    "w-full px-4 py-4 text-center text-xs font-bold tracking-widest transition-colors",
                                    option.value === value
                                        ? "bg-primary text-white"
                                        : "text-obsidian hover:bg-muted/30"
                                )}
                            >
                                {option.label}
                            </button>
                        ))}
                    </div>
                )}
            </div>
        </div>
    );
}
```

</AccordionGroup>

## ProjectList

A table component that renders a list of projects with loading skeletons and empty states. Accepts optional `onView` and `onEdit` callbacks to surface action buttons per row.

**Props**

| Prop | Type | Description |
|---|---|---|
| `projects` | array | Array of project objects to render |
| `loading` | boolean | When `true` while `hasData` is false, shows a spinner; when `true` with data, shows skeleton rows |
| `hasData` | boolean | Whether data has been loaded at least once |
| `onView` | `(project) => void` | If provided, renders a "Ver" button per row |
| `onEdit` | `(project) => void` | If provided, renders an "Editar" button per row |

The component renders three states:
- **Loading (no data)** — centered spinner with "Sincronizando..." text
- **Empty** — centered briefcase icon with "Sin registros" text
- **Populated** — table with skeleton rows overlaid during background refreshes

## ErrorModal

A fixed bottom-right toast notification triggered by the error simulation system. Color-coded by HTTP status code.

**Props**

| Prop | Type | Description |
|---|---|---|
| `isOpen` | boolean | Whether the modal is visible |
| `onClose` | `() => void` | Called when the user clicks the close button |
| `errorCode` | number | HTTP status code that determines the color and label |
| `message` | string | Body text shown inside the modal |

**Color coding**

| `errorCode` | Background | Label |
|---|---|---|
| `200` | Emerald | Protocolo Restablecido |
| `401` | Violet | Acceso no Autorizado |
| `404` | Blue | Recurso no Encontrado |
| `500` | Red | Falla Crítica del Sistema |
| other | Amber | Aviso de Auditoría |

When `errorCode` is `200`, the modal auto-dismisses after 2.5 seconds and displays a shrinking progress bar.

<AccordionGroup>
  <Accordion title="ErrorModal usage example">

```jsx
    isOpen={errorModal.isOpen}

    errorCode={errorModal.code}
    message={errorModal.message}
/>
```

  </Accordion>
  <Accordion title="ErrorModal source code">

```jsx
            bg: "bg-blue-600",
            border: "border-blue-700",
            icon: <Info size={20} className="text-white" />,
            label: "Recurso no Encontrado"
        },
        200: {
            color: "text-white",
            bg: "bg-emerald-500",
            border: "border-emerald-600",
            icon: <CheckCircle2 size={20} className="text-white" />,
            label: "Protocolo Restablecido"
        },
        401: {
            color: "text-white",
            bg: "bg-violet-600",
            border: "border-violet-700",

            label: "Aviso de Auditoría"

    return (

            <div className={cn(

                {/* ... icon, label, message, close button */}
            </div>

}
```

## Layout

The root application shell. Renders the sidebar, sticky header, and main content area. Manages sidebar collapse state and the engineering panel visibility internally.

**Props**

| Prop | Type | Description |
|---|---|---|
| `children` | ReactNode | Page content rendered in the main area |
| `user` | `{ name, role }` | Used to render the user's name and translated role in the header |
| `activeTab` | string | Currently active navigation tab key |
| `setTab` | `(tab: string) => void` | Called when a sidebar nav item is clicked |

**Features**

- **Collapsible sidebar** — a toggle button at the bottom of the sidebar collapses it to icon-only mode (`w-20`). Tooltip labels appear on hover in collapsed mode.
- **Sticky header** — the top bar stays fixed while the main content scrolls. It shows the current user's name and a generated avatar from DiceBear.
- **Engineering panel** — the **Control de Red** section is rendered only when the sidebar is expanded (`!isCollapsed`). It exposes three error injection buttons and a restore button that call `window.setSimulatedError`.
- **Tab navigation** — clicking Dashboard, Proyectos, or Usuarios calls `setTab`, which `App.jsx` uses to conditionally render the corresponding page component.

<AccordionGroup>
  <Accordion title="Layout usage example">

```jsx

// In App.jsx
const user = { name: "Ing. Wilber", role: "Engineering Lead" };

<Layout user={user} activeTab={activeTab} setTab={setActiveTab}>
    {activeTab === 'dashboard' && <DashboardPage />}
    {activeTab === 'projects' && <ProjectsPage />}
    {activeTab === 'users' && <UsersPage />}
</Layout>
```
