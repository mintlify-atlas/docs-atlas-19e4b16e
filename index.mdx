## What is APM Enterprise?

APM Enterprise is a React-based management dashboard designed for engineering teams. It provides real-time KPI tracking, project portfolio management, and a user directory — all backed by a centralized HTTP client with Bearer token authentication and a built-in mock server for local development.

<Info>
  APM Enterprise uses a **Separation of Concerns (SoC)** architecture that cleanly separates the presentation layer (pages and components), business logic layer (custom hooks), and data layer (services and HTTP client).
</Info>

## Key features

<AccordionGroup>
  <Accordion title="KPI dashboard with live data sync">
    The dashboard displays total revenue, new users (last 7 days), and churn rate. Data is fetched from the Enterprise API via `apiGetDashboard` and automatically re-fetches when an `apm:api_status` event fires — ensuring the UI stays in sync after any error simulation or restore.
  </Accordion>
  <Accordion title="Project management with CRUD">
    Create new projects with name, owner, budget, and status. Edit existing projects inline. Filter by status (active / paused) and browse pages. All mutations go through the centralized `httpClient` with Bearer auth.
  </Accordion>
  <Accordion title="User directory with audit traceability">
    Lists users from JSONPlaceholder with paginated table view. Selecting a user triggers deep audit logging in the browser console — fetching their posts and top comments to demonstrate network traceability.
  </Accordion>
  <Accordion title="API engineering panel with error simulation">
    The sidebar contains a hidden engineering panel that lets you inject HTTP error codes (401, 404, 500) globally using `window.setSimulatedError(code)`. The app reacts immediately — clearing data, showing error modals, and restoring on 200.
  </Accordion>
  <Accordion title="Layered, maintainable architecture">
    Pages consume custom hooks (`useDashboard`, `useProjectForm`, `useUsers`) that abstract all async state. Hooks call service functions that use `httpClient`. No page component ever calls `fetch` directly.
  </Accordion>
</AccordionGroup>

## Technology stack

| Layer | Technology |
|---|---|
| UI Framework | React 18 |
| Build Tool | Vite 5 |
| Styling | Tailwind CSS v4 |
| Icons | Lucide React |
| HTTP | Native `fetch` via centralized `httpClient` |
| State | React hooks (`useState`, `useMemo`, `useEffect`) |
| Mock API | Custom `mockServer.js` fetch interceptor |

## Getting started

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/CHANKUN5/APM.git
    cd APM
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash npm
    npm install
    ```
  </Step>
  <Step title="Start the development server">
    ```bash
    npm run dev
    ```
    Open [http://localhost:5173](http://localhost:5173) in your browser.
  </Step>
  <Step title="Log in to the dashboard">
    Use the demo credentials pre-filled on the login screen:
    - **Email:** `admin@demo.com`
    - **Password:** `123456`

    Click **Iniciar Protocolo** to access the dashboard.
  </Step>
</Steps>
