<Tip>
  APM Enterprise runs entirely in demo mode. The mock server intercepts all HTTP calls to `api.apm-enterprise.com` — no real backend or network connection is required.
</Tip>

## Prerequisites

- **Node.js** 18 or higher
- **npm**, **yarn**, or **pnpm**

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/CHANKUN5/APM.git
    cd APM
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash npm
      npm install
      ```
      ```bash yarn
      yarn install
      ```
      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Vite starts the dev server. Open [http://localhost:5173](http://localhost:5173) in your browser.
  </Step>

  <Step title="Log in to the dashboard">
    The app opens on the login screen. Use the pre-filled demo credentials:

    - **Email:** `admin@demo.com`
    - **Password:** `123456`

    Click **Iniciar Protocolo** to authenticate and enter the dashboard.

    <Note>
      The login validation requires the email to contain `@` and the password to be at least 3 characters long. The token is hardcoded to `"demo-token"` — no real auth server is involved.
    </Note>
  </Step>

  <Step title="Explore the modules">
    Use the left sidebar to navigate between the three active modules:

    | Module | Sidebar label | Description |
    |---|---|---|
    | Dashboard | Dashboard | KPI stats, project list, create/edit form |
    | Projects | Proyectos | Full project inventory with pagination |
    | Users | Usuarios | External user directory with audit trail |

    <Warning>
      The **Configuración** tab is currently reserved and shows a placeholder message. This module is not yet implemented.
    </Warning>
  </Step>
</Steps>

## Available scripts

| Script | Command | Description |
|---|---|---|
| Development | `npm run dev` | Starts Vite dev server with HMR |
| Build | `npm run build` | Produces a production bundle in `dist/` |
| Preview | `npm run preview` | Serves the production build locally |

## Try the API engineering panel

The sidebar includes a built-in error simulation panel. To use it:

1. Make sure the sidebar is expanded (not collapsed).
2. Click **Control de Red** in the sidebar to expand the panel.
3. Click **500**, **401**, or **404** to inject a simulated error globally.
4. Click **Restaurar (200)** to restore normal operation.

You can also drive it from the browser console:

```js
// Inject error codes
window.setSimulatedError(500)  // Triggers a 500 Core Failure
window.setSimulatedError(401)  // Triggers a 401 Auth Failure
window.setSimulatedError(404)  // Triggers a 404 Not Found

// Restore normal operation
window.setSimulatedError(null)
```

<Info>
  When you call `window.setSimulatedError(code)`, it dispatches a global `apm:api_status` CustomEvent. All active hooks re-execute their fetches, which now return the simulated error code from the mock server.
</Info>
